# Linksee Chat 产品实现清单

本清单用于约束后续迭代顺序。状态说明：`完成` 表示当前代码已有可用实现，`进行中` 表示已有基础但仍需增强，`待做` 表示尚未进入正式实现，`暂缓` 表示首版不做或依赖后续架构。

## 设计约束

- 客户端定位：类 QQ 的桌面聊天客户端，不把服务器地址、调试日志、协议细节暴露给普通用户。
- 服务端定位：认证、会话、消息、文件、资料、更新统一由后端提供，客户端只做展示、缓存和交互。
- 工程约束：常规文件尽量控制在 100 行左右，复杂组件拆分；单文件超过 500 行必须优先拆分；任何文件不得超过 1000 行。
- 设计模式：新功能开发前先明确边界，优先使用组合、策略、外观、适配器、观察者等模式降低后续返工。
